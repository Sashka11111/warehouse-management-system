# Warehouse Management System

Система управління складом (Warehouse Management System) — це настільний застосунок (desktop application), розроблений на Java з використанням JavaFX. Він забезпечує зручний, сучасний та професійний інтерфейс для управління запасами, зонами зберігання, категоріями товарів та іншими аспектами складського обліку.

## ✨ Основні можливості (Features)

- **Модернізований Dashboard (Головна панель):** зручний перегляд ключових фінансових та складських метрик у реальному часі.
- **Управління зонами (Zone Management):** відстеження розташування товарів за допомогою сучасної системи "Зон" (Zone tracking system).
- **Категорії товарів:** створення, редагування та видалення категорій з надійною валідацією даних.
- **Фільтрація та пошук:** швидкий пошук сутностей (Entity Search Filters) для зручної навігації великими обсягами даних.
- **Сучасний UI/UX:** шрифти Segoe UI та загальна стилістика, орієнтована на професійне використання.

## 🛠 Технологічний стек

- **Мова:** Java 26
- **UI Фреймворк:** JavaFX 26
- **Тематика та компоненти UI:**
  - [AtlantaFX](https://mkpaz.github.io/atlantafx/) (сучасна система стилів для JavaFX)
  - Ikonli (іконки FontAwesome 5)
- **База даних:** SQLite (`sqlite-jdbc`)
- **Пул з'єднань:** HikariCP
- **Логування:** SLF4J
- **Збирання проєкту:** Maven

## 🚀 Встановлення та запуск

### 📦 Завантаження (Для користувачів)

Найпростіший спосіб почати роботу — завантажити готовий інсталятор для вашої операційної системи зі сторінки **[Releases](../../releases)** на GitHub.

- **Windows:** завантажте та запустіть файл `.msi`.
- **macOS:** завантажте файл `.dmg` та перетягніть програму до теки *Applications*.
- **Linux:** використовуйте пакети `.deb` (для Debian/Ubuntu) або `.rpm` (для Fedora/RedHat).
- **Кросплатформенно:** також доступний файл `.jar` (вимагає попередньо встановленої Java 26).

### 💻 Для розробників (Збірка з коду)

Якщо ви бажаєте допрацьовувати проєкт або зібрати його власноруч, вам знадобляться:
- **Java Development Kit (JDK) 26**
- **Apache Maven**

**1. Клонування репозиторію:**
```bash
git clone <url-репозиторію>
cd warehouse-management-system
```

**2. Швидкий запуск:**
Завдяки плагіну `javafx-maven-plugin` ви можете запустити програму однією командою:
```bash
mvn clean javafx:run
```

**3. Збирання виконуваного JAR-файлу (Fat JAR):**
Проєкт налаштовано на створення єдиного JAR-файлу з усіма залежностями за допомогою `maven-shade-plugin`.
```bash
mvn clean package -DskipTests
```
Після успішної збірки запустіть його командою:
```bash
java -jar target/warehouse-management-system-1.0-SNAPSHOT.jar
```

## 🗄 База даних

Проєкт використовує **SQLite**. База даних (зазвичай локальний файл `.db`) створюється/знаходиться локально (перевірте папку `db` у корені проєкту). Підключення та управління пулом з'єднань здійснюється за допомогою HikariCP для високої продуктивності.
